# Security invariants (do not weaken)

- **Dual-path matching** (`src/match.ts`): every request is matched on the raw pathname AND (if different) its canonical form. The canonical pass may **add or override — never delete** — a rule the raw path resolved, and each path's `false` reset is resolved independently before the union. This blocks encoded-separator bypasses.
- **`canonicalPath` / `isPathInScope`** (`src/internal/scope.ts`): canonical results are for rule matching and scope checks only — never for routing/dispatch or proxy/redirect forwarding. Proxy/redirect forward the **raw** `event.url.pathname` (encoded `%2f`/`%5c` stay opaque to the upstream). `isPathInScope` fails closed under **two** interpretations: h3's empty-segment-preserving canonical form AND a separator-run-collapsed form (`SEPARATOR_RUN_RE`, mirroring h3's `%2f`/`%5c` decode). They diverge when a `..` sits next to an empty `//` segment — h3 lets the empty shield the `..` (in scope), but a slash-merging downstream (nginx `merge_slashes`) drops the empty and the `..` traverses out; requiring the collapsed reading in scope too closes that gap (covers literal `//`, encoded `%2f%2f`, and nested variants). `isPathInScope` is scope-only (not used by matching), so this stays strict-fail-closed without affecting dispatch.
- **Wildcard redirect/proxy scope**: for `base`-scoped rules, out-of-scope canonical traversal must throw 400 — and so must any request whose **raw** path does not literally sit under the base (encoded separator inside the base region, canonical-only match via dot segments): the base cannot be faithfully stripped from the raw path, so `resolveRuleTarget` fails closed rather than forwarding it unstripped. Base-less `//` prefixes must be collapsed, never forwarded as protocol-relative URLs.
- **Final-target scope check**: scope is enforced on the **resolved upstream target** (after base strip + `joinURL`), not only the incoming path. A doubled/leading slash or `/./` can make the incoming path canonicalize _inside_ base (an empty `//` segment absorbing a `..`), yet once the base is stripped and the remainder rejoined — `joinURL` collapses the empty segment — the `..%2f` escapes the target's own base the moment a downstream decodes it. The check re-canonicalizes the final target's pathname against the target base (the `to` minus `/**`) and throws 400 on escape. Pinned by the "resolveRuleTarget final-target scope" block and the doubled-slash case in "encoded-separator hardening" (`test/rules.test.ts`).
- **`basicAuth` has `order: -2`** (outer to `headers` at -1 and everything else at 0) so unauthorized requests are never redirected/proxied/cached and never carry `headers` (auth-vs-redirect/proxy pinned in `test/rules.test.ts`, auth-vs-cache in `test/cache.test.ts`).
- The regression suites in `test/scope.test.ts`, the "dual-path union" block in `test/merge.test.ts`, and the "encoded-separator hardening" block in `test/rules.test.ts` pin all of the above — do not weaken or delete them.
